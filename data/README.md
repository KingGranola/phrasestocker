# データとバックアップの管理

このフォルダは、PhraseStocker のデータを安全に保管するために使用されます。

## 📁 フォルダ構成

```
phrasestocker/
├── data/           ← 保存したフレーズデータ（.json）
├── backups/        ← 自動バックアップ
└── README.md
```

## 💾 データの保存方法

### 方法1: アプリ内でエクスポート（推奨）

1. アプリを起動
2. サイドバーのライブラリから保存したいフレーズを選択
3. メニューから「エクスポート」→「個別ファイル」を選択
4. `data/` フォルダに保存

### 方法2: ライブラリ全体をエクスポート

1. アプリを起動
2. メニューから「エクスポート」→「ライブラリ全体」を選択
3. `data/` フォルダに `phrasestocker-library-<日付>.json` として保存

## 📥 データの読み込み方法

1. アプリを起動
2. メニューから「インポート」を選択
3. `data/` フォルダから読み込みたいファイルを選択

## 🔄 自動バックアップ

アプリは1日1回、自動的にバックアップを作成します：

- **保存場所**: LocalStorage（ブラウザ内）
- **手動バックアップ**: メニューから「バックアップ作成」を選択

推奨: 定期的に手動でエクスポートして、`backups/` フォルダに保存してください。

## ⚠️ 注意事項

### データの互換性

- バージョンアップ後も、古いデータは読み込み可能です
- ただし、新機能を使用したデータは古いバージョンで開けない場合があります

### バックアップの重要性

LocalStorageは以下の場合にクリアされる可能性があります：
- ブラウザのキャッシュクリア
- ブラウザの再インストール
- ストレージ容量不足

**重要なデータは必ず `data/` フォルダにエクスポートしてください！**


## 🔧 トラブルシューティング

### データが消えた場合

1. `backups/` フォルダの最新ファイルを確認
2. 「インポート」機能で復元
3. それでも見つからない場合、LocalStorageのバックアップを確認：
   - ブラウザの開発者ツール → Application → Local Storage → `phraseStocker_backup`

### ファイルが開けない場合

- ファイルが壊れている可能性があります
- 別のバックアップファイルを試してください
- テキストエディタで開いて、JSON形式が正しいか確認

## 🎯 ベストプラクティス

1. **毎週バックアップ**: 週に1回、ライブラリ全体をエクスポート
2. **重要なフレーズ**: すぐに個別ファイルとして保存
3. **プロジェクト別**: プロジェクトごとにフォルダを分ける
4. **バージョン管理**: Gitで `data/` フォルダを管理（オプション）
